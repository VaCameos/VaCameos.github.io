# $set

this.$set(target,key,value)

  • 第一个参数是修改的对象
  • 第二个参数是要修改的key
  • 第三个是要修改的值

有没有数据修改了视图没有修改的情况？

data(){
    return {
        arr:[1,2,3]
    }
}
methods:{
    btn(){
        this.arr[1]='xxx' // 此时数据更改了但视图不会改
        this.$set(this.arr,'1','xxx')
    }
}

# data定义数据

数据定义在data的return内和return外的区别：

  • return外：单纯修改这个数据是不可修改的，因为没有get/set
  • return内：可修改

# data为什么是一个函数

  • data是一个函数，这样每次复用一次组件，就会返回一份新的data ，（相当于每个组件实例创建私有空间，各自维护自己的数据
  • object是引用类型，保存的是内存地址，data写成对象形式，就使得所有组件实例共用一份data，例如一个组件被复用3次 ，当某一处复用的地方组件内data数据被改变时就会影响到其他地方

# computed

data(){
    return{
        test:'21'
    }
}


// 1 修改会报错 has no setter
computed:{
    str(){
        return test+'1'
    }
}
// 2 可以完成修改
computed:{
    str{
        get(){
            return test+'1'
        } ,
        set(val){
             this.test = val
            // 触发set后触发get
        }
    }
}
this.changeStr = 'xxx'

  • computed计算属性的结果值，可以修改吗？

    答：可以，需要通过get/set写法

  • 当前组件v-model绑定的值是computed来的，可以修改吗? 可以，需要通过get/set写法

# 如何自定义指令

  • 全局：main.js
Vue.directive('demo',{
    inserted:function(a,b,c){
        console.log(a,b,c)
    }
})
  • 局部：某个组件内
export default {
    directives{
    	demo:{
			bind:function(el){
                console.log(el)
            }    
		}
	}
}
