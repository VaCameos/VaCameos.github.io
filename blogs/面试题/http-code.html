ray2023-02-28面试题http
  • 1xx表示消息
  • 2xx表示成功
  • 3xx表示重定向
  • 4xx客户端错误
  • 5xx服务端错误

# 常见的状态码

# 101 Switch Protocol

升级协议，http到ws，需要反向代理支持，如nginx

# 200 OK

表示资源请求成功，最常见的状态码

# 201 Created

资源创建成功，多用于post请求

# 204 No Content

响应不会返回Body，一般是以下两种情况

  • 与Options/Delete 请求搭配
  • 打点类

# 206 Partial Content

分片传输，当请求多媒体数据较大时会进行分片传输，例如b詹视频

# 301 Moved Permanently

永久重定向。 http转https有时会用到

# 302 Found

暂时重定向

# 304 Not Modified

资源已被缓存，与之相关的响应头

  • ETag
  • last-modified/if-modified-since

一般用作 index.html 等不带 hash 的资源，由于示例太多，这里就不举例了

与协商缓存 和强缓存有关

# 307 Temporary Redirect

暂时重定向。也可作为 http 到 https 的重定向。还有一种用途用作 HSTS，当谷歌浏览器发现某 http 资源已被加入到 HSTS 列表，浏览器内部会通过 307 作重定向

# 301、302、307的区别

  • 301 比较危险，如果设置了301想取消，但是浏览器中已经有了缓存，还是会重定向，且SEO只和重定向的网站有关

  • 302 SEO对两个网站都有效 会把method改变：POST-> GET ,因此出现307

  • 和302类似 但不会改变method

# 400 Bad Request

对于服务器无法理解的参数，将会使用400作为返回码

示例：当Content-Type：JSON时 ，服务器解析JSON失败

HTTP/1.1 400 Bad Request
Content-Length: 35

{"message":"Problems parsing JSON"}

# 401 Unauthorized

没权限的用户请求需要带权限的资源时 ， 有时认证失败也会返回401

例：登录时密码不正确

# 403 Forbidden

禁止访问，

# 404 Not Found

未找到资源

# 405 Method Not Allowed

method错误

# 413 Payload Too Large

负载过大 body过大

# 500 Internal Server Error

服务器内部错误，很有可能是应用层未捕获错误而导致整个服务挂掉

# 502 Bad Gateway

Nginx常见，从上游应用层未返回响应，上游应用层挂了

# 503 Service Unavailable

服务器繁忙

# 504 Gateway Timeout

网关超时
